# 查询

在配置好元数据之后，我们就可以开始着手构建查询了，sqala的基础查询结构与Scala集合库非常相似：

import sqala.compiletime.*

val q = query(tablePeople).filter(_.id === 1).map(p => (p.id, p.name))

如果是SELECT ... FROM ... WHERE这样常见的查询，我们也可以使用Scala的for推导式来构建：

import sqala.compiletime.*

val q = for p <- query(tablePeople) if p.id === 1 yield (p.id, p.name)

# 构建查询

sqala的查询使用query方法构建，参数是使用asTable生成的表：

val q = query(tablePeople)

其生成的SQL（以MySQL方言为例）形如：

SELECT `people`.`id`, `people`.`name`, `people`.`gender`, `people`.`birthday`, `people`.`id_number` FROM `people`

在配置好数据库信息之后，查询返回的数据为：

val result: List[People] = db.select(q)

# 投影

使用map方法进行结果集的投影操作，其对应到SQL的SELECT子句，参数是一个表结构对象映射到表达式 (opens new window)或表达式组成的元组：

val q = query(tablePeople).map(p => (p.id, p.name))

其生成的SQL（以MySQL方言为例）形如：

SELECT `people`.`id`, `people`.`name` FROM `people`

我们也可以将上面的写法转化为Scala的for推导式：

val q = for p <- query(tablePeople) yield (p.id, p.name)

查询返回的数据为：

val result: List[(Int, String)] = db.select(q)

# 过滤

使用filter方法进行条件过滤，其对应到SQL的WHERE子句，参数是一个Boolean表达式 (opens new window)：

val id = 1
val q = query(tablePeople).filter(_.id === id).map(p => (p.id, p.name))

其生成的SQL（以MySQL方言为例）形如：

SELECT `people`.`id`, `people`.`name` FROM `people` WHERE `people`.`id` = ?

我们也可以在for推导式中使用if来调用它：

val q = for p <- query(tablePeople) if p.id === 1 yield (p.id, p.name)

多次调用filter时将会使用AND来连接查询条件。

对于动态拼接的条件，sqala提供了filterIf和filterNot两个方法，前者会在第一个参数值为true时使用AND才将条件拼接到查询中，后者与其相反，在第一个参数值为false时拼接：

val id = 1
val name = "小黑"

val q = query(tablePeople).filterIf(id > 0)(_.id === id).filterNot(name.isEmpty)(_.name === name)

filter不会改变查询结果类型。

# 表连接

sqala支持使用join、leftJoin、rightJoin、fullJoin来处理表连接，并使用on来处理链接条件：

val q = query(tableA join tableB on ((a, b) => a.id === b.id) join tableC on ((_, b, c) => b.id === c.id))

on引用的参数类型为一个由表对象组成的元组，元组项的数量会根据连接的表数量变化。

此时返回的结果类型为：

val result: List[(A, B, C)] = db.select(q)

由于外连接会产生额外的空值，因此，sqala可以从表连接的信息中自动推断返回类型，如果我们把上面的查询改为先进行右连接再进行左连接：

val q = query(tableA rightJoin tableB on ((a, b) => a.id === b.id) leftJoin tableC on ((_, b, c) => b.id === c.id))

此时返回的结果类型变成了：

val result: List[(Option[A], B, Option[C])] = db.select(q)

这里显式写出了返回类型仅仅是为了演示，实际上sqala总会从查询中自动推断出合适的返回类型。

另外，我们也可以对表对象使用as方法起别名，来处理连接自身的情况，比如我们有一个存储部门树的表对象：

case class Department(
    id: Int,
    parentId: Int,
    name: String
) derives Entity
val tableDepartment = asTable[Department]

我们可以这样来关联查询：

val t1 = tableDepartment as "t1"
val t2 = tableDepartment as "t2"
val q = for (a, b) <- query(t1 leftJoin t2 on (_.parentId === _.id)) yield (a.name, b.name)

# 分组

使用groupBy配合聚合函数表达式进行分组操作：

val q = query(tablePeople).groupBy(p => (p.name, p.gender)).map(((n, g), p) => (n, g, count, max(p.birthday)))

其生成的SQL（以MySQL方言为例）形如：

SELECT `people`.`name`, `people`.`gender`, COUNT(*), MAX(`people`.`birthday`) FROM `people` GROUP BY `people`.`name`, `people`.`gender`

另外可以使用having过滤聚合出的数据：

val q = query(tablePeople).groupBy(p => (p.name, p.gender)).having(_ => count > 1).map(((n, g), p) => (n, g, count, max(p.birthday)))

其生成的SQL形如：

SELECT `people`.`name`, `people`.`gender`, COUNT(*), MAX(`people`.`birthday`) FROM `people` GROUP BY `people`.`name`, `people`.`gender` HAVING COUNT(*) > 1

# 排序

使用sortBy方法进行排序，参数是使用asc或desc方法生成的排序规则，多个sortBy会依次拼接：

val q = query(tablePeople).sortBy(_.id.asc).sortBy(_.name.desc)

其生成的SQL（以MySQL方言为例）形如：

SELECT `people`.`id`, `people`.`name`, `people`.`gender`, `people`.`birthday`, `people`.`id_number` FROM `people` ORDER BY `people`.`id` ASC, `people`.`name` DESC

sortBy不会改变查询结果类型

# 去重

使用distinct方法来对结果集进行去重：

val q = query(tablePeople).map(_.name).distinct

# 子查询

sqala支持表达式 (opens new window)中的子查询和FROM子句中的表子查询，首先来介绍表达式中的子查询:

子查询可以作为表达式的一部分出现，比如：

val subQuery = for p <- query(tablePeople) yield max(p.birthday)

val q = for p <- query(tablePeople) if p.birthday < subQuery yield p.id

其生成的SQL（以MySQL方言为例）形如：

SELECT `people`.`id` FROM `people` WHERE `people`.`birthday` < (
    SELECT MAX(`people`.`birthday`) FROM `people`
)

此外也支持IN、ANY、SOME、ALL、EXISTS等子查询相关谓词：

val subQuery = for p <- query(tablePeople) if p.name === "小黑" yield p.id

val q1 = query(tablePeople).filter(_.id === any(subQuery))
val q2 = query(tablePeople).filter(_.id.in(subQuery))

FROM中的子查询，需要我们使用as方法给字段和查询起别名，然后再次引用：

val subQuery = for a <- query(tablePeople) yield (a.id as "col1", a.name as "col2") as "q1"

val q = for q1 <- query(subQuery) yield (q1.col1, q1.col2)

sqala将会自动从字段的别名中生成字段，我们可以使用.来调用生成出的字段来构建查询。

表子查询也可以使用join等方法连接其他表，不再赘述用法。

# 集合操作

sqala支持使用union、unionAll、intersect、intersectAll、except、exceptAll等方法来处理集合查询，比如：

val q1 = for p <- query(tablePeople) if p.id === 1 yield (p.id, p.name)
val q2 = for p <- query(tablePeople) if p.id === 2 yield (p.id, p.name)
val q = q1 union q2

由于UNION ALL与集合拼接语义一致，所以unionAll可以简写成++：

val q = q1 ++ q2

处于集合操作两侧的查询，其返回类型必须列数量一致，且类型一一对应，假如有两个查询，分别返回(Option[Int], String, Option[Date])和(Int, Option[String], Date)，这样的两个查询调用集合操作将会返回(Option[Int], Option[String], Option[Date])。

# 递归查询

回顾一下在表连接部分提到的部门表：

case class Department(
    id: Int,
    parentId: Int,
    name: String
) derives Entity
val tableDepartment = asTable[Department]

我们想查询一个部门下面的所有层级的下级部门时，如果分开多次查询，开销比较高，如果不确定下级部门的层级数目，代码编写将会比较繁琐，所幸数据库支持使用CTE(common table expression)语句来处理递归查询，我们可以编写这样的SQL（以MySQL为例）：

WITH RECURSIVE `tmp` as (
    SELECT * FROM `department` WHERE `department`.`parent_id` = 0
    UNION ALL
    SELECT `department`.* FROM `department` INNER JOIN `tmp` ON `department`.`parent_id` = `tmp`.`id`
)
SELECT * FROM `tmp`

sqala也支持编写这样的结构，我们使用withRecursive来创建查询：

val baseQuery = for d <- query(tableDepartment) if d.parentId === 0 yield (d.id, d.name)

val cte = withRecursive(baseQuery, "tmp"): (q, tmp) =>
    val unionQuery = for (dept, _) <- query(tableDepartment join tmp on (_.parentId === _.id)) yield (dept.id, dept.name)
    q ++ unionQuery
.select: dept =>
    for d <- query(dept) yield (d.id, d.name)

# 查询

在配置好元数据之后，我们就可以开始着手构建查询了，sqala使用类似Scala集合库风格的api创建查询。

以下用法均需要导入：

import sqala.dsl.*

# 构建查询

sqala的查询使用query方法构建，其类型参数是实体类的名称：

val q = query[People]

其生成的SQL（以MySQL方言为例）形如：

SELECT
    `t0`.`id` AS `c0`,
    `t0`.`name` AS `c1`,
    `t0`.`gender` AS `c2`,
    `t0`.`birthday` AS `c3`,
    `t0`.`id_number` AS `c4`
FROM `people` AS `t0`

查询返回的数据类型为：

val result: List[People] = db.fetch(q)

# 投影

使用map方法进行结果集的投影操作，其对应到SQL的SELECT子句，参数是一个表结构对象映射到表达式 (opens new window)及其组成的元组或命名元组：

# 投影到表达式

val q = query[People].map(_.id)

其生成的SQL（以MySQL方言为例）形如：

SELECT
    `t0`.`id` AS `c0`
FROM `people` AS `t0`

查询返回的数据为：

val result: List[Int] = db.fetch(q)

# 投影到元组

val q = query[People].map(p => (p.id, p.name))

其生成的SQL（以MySQL方言为例）形如：

SELECT
    `t0`.`id` AS `c0`,
    `t0`.`name` AS `c1`
FROM `people` AS `t0`

查询返回的数据为：

val result: List[(Int, String)] = db.fetch(q)

# 投影到命名元组

投影到命名元组可以更加明确查询的目的，并且在返回结果集中可以使用.直接调用其中的字段，而无需另行创建实体类接收中间结果。使用命名元组之前，我们需要确保开启了编译器选项：-experimental，并在调用处导入import scala.language.experimental.namedTuples。

import sqala.dsl.*

import scala.language.experimental.namedTuples

val q = query[People].map(p => (id = p.id, name = p.name))

查询返回数据为：

val result: List[(id: Int, name: String)] = db.fetch(q)

我们可以直接使用.调用其中字段：

for r <- result do
    println(r.id)
    println(r.name)

使用方法*可以将表快速投影到对应的命名元组，在子查询等情况下有重要作用：

val q = query[People].map(p => p.*)

# 投影的限制

在未分组的查询中，如果投影中同时出现了聚合函数和非聚合函数表达式，如：

val q = query[People].map(p => (p.id, count()))

将会得到编译错误。sqala可以在编译期拦截这样错误语义的查询。

# 过滤

使用filter方法进行条件过滤，其对应到SQL的WHERE子句，参数是一个Boolean类型的表达式 (opens new window)：

val id = 1
val q = query[People].filter(_.id == id).map(p => (p.id, p.name))

其生成的SQL（以MySQL方言为例）形如：

SELECT
    `t0`.`id` AS `c0`,
    `t0`.`name` AS `c1`
FROM `people` AS `t0`
WHERE `t0`.`id` = ?

多次调用filter时将会使用AND来连接查询条件。

对于动态拼接的条件，sqala提供了filterIf方法，会在第一个参数值为true时使用AND才将条件拼接到查询中：

val id = 1
val name = "小黑"

val q = query[People].filterIf(id > 0)(_.id == id).filterIf(name.nonEmpty)(_.name == name)

filter不会改变查询结果类型。

# 过滤条件的限制

在filter中的表达式如果包含聚合函数、窗口函数等情况，将产生编译错误：

// 编译错误
val q = query[People].filter(p => count() > 0)

# 使用for推导式

只有过滤和投影操作的简单查询可以使用Scala的for推导式简化，比如以下查询：

val q = query[People].filter(_.id == 1).map(_.name)

可以简写成：

val q = for p <- query[People] if p.id == 1 yield p.name

通常开发中最常使用的就是简单的查询，推荐写成for的形式以获取更好的可读性。

# 表连接

sqala支持使用join、leftJoin、rightJoin来处理表连接，并使用on来处理连接条件：

val q =
    query[A]
        .join[B]
        .on((a, b) => a.id == b.id)
        .join[C]
        .on((a, b, c) => b.id == c.id)

on引用的参数类型为一个由表对象组成的元组，元组项的数量会根据连接的表数量变化。

此时返回的结果类型为：

val result: List[(A, B, C)] = db.fetch(q)

.on可以省略，上面的查询可以简写成：

val q =
    query[A]
        .join[B]((a, b) => a.id == b.id)
        .join[C]((a, b, c) => b.id == c.id)

由于外连接会产生额外的空值，因此，sqala可以从表连接的信息中自动推断返回类型，如果我们把上面的查询改为先进行右连接再进行左连接：

val q =
    query[A]
        .rightJoin[B]((a, b) => a.id == b.id)
        .leftJoin[C]((a, b, c) => b.id == c.id)

此时返回的结果类型变成了：

val result: List[(Option[A], B, Option[C])] = db.fetch(q)

这里显式写出了返回类型仅仅是为了演示，sqala会从查询中自动推断出合适的返回类型。

我们也可以方便地处理连接自身的情况，比如有一个存储部门树的表对象：

case class Department(
    id: Int,
    parentId: Int,
    name: String
)

我们可以这样来关联查询：

val q = query[Department].leftJoin[Department]((t1, t2) => t1.parentId == t2.id)

除了.on之外，我们可以使用.using创建连接条件，其允许我们简化同名字段相等的连接条件：

val q = query[A].join[B].using((a, _) => a.id)

# 分组

使用groupBy配合聚合函数表达式进行分组操作，在groupBy之后的map中，有两个参数，第一个是分组字段列表，第二个是原表。如果对多个字段分组，此时的类型可能比较复杂，推荐使用命名元组分组：

val q =
    query[People]
        .groupBy(p => (name = p.name, gender = p.gender))
        .map: (g, p) =>
            (g.name, g.gender, count(), max(p.birthday))

其生成的SQL（以MySQL方言为例）形如：

SELECT
    `t0`.`name` AS `c0`,
    `t0`.`gender` AS `c1`,
    COUNT(*) AS `c2`,
    MAX(`t0`.`birthday`) AS `c3`
FROM `people` AS `t0`
GROUP BY `t0`.`name`, `t0`.`gender`

分组使用有如下限制：

  1. 分组后的投影，必须是分组字段或聚合函数表达式。
  2. 分组表达式不能包含聚合函数或窗口函数。
  3. 分组表达式不能包含常量表达式。

以上情况均会产生编译错误。

在投影之前可以使用having过滤聚合出的数据：

val q =
    query[People]
        .groupBy(p => (name = p.name, gender = p.gender))
        .having(_ => count() > 1)
        .map: (g, p) =>
            (g.name, g.gender, count(), max(p.birthday))

除了简单的表达式分组外，我们也可以使用groupByCube和groupByRollup两种分组形式，使用方法与普通的groupBy类似：

val q1 =
    query[People]
        .groupByCube(p => (name = p.name, gender = p.gender))
        .map: (g, p) =>
            (g.name, g.gender, count(), max(p.birthday))

val q2 =
    query[People]
        .groupByRollup(p => (name = p.name, gender = p.gender))
        .map: (g, p) =>
            (g.name, g.gender, count(), max(p.birthday))

我们也可以使用groupByGroupingSets自行指定多个分组：

val q =
    query[People]
        .groupByGroupingSets(p => (name = p.name, gender = p.gender))(g => (g.name, g.gender, (g.name, g.gender), EmptyTuple))
        .map: (g, p) => 
            (g.name, g.gender, count(), max(p.birthday))

groupByGroupingSets的第一个参数与普通的groupBy形式类似，用于指定基础分组，第二个参数是第一个参数的返回结果组织成的元组，空分组使用EmptyTuple指定。

在使用groupByCube、groupByRollup和groupByGroupingSets这样较为复杂的分组时，我们可以在投影中使用grouping查看分组处理情况，对应到SQL的GROUPING表达式：

val q = query[A]
    .groupByCube(a => (x = a.x, y = a.y))
    .map: (g, a) =>
        (
            g.x,
            grouping(g.x),
            g.y,
            grouping(g.y),
            sum(a.z)
        )

# 排序

在投影后，我们可以使用sortBy方法进行排序，参数是表达式的排序规则，多个sortBy会依次拼接：

val q = query[People].map(_.*).sortBy(_.id.asc).sortBy(_.name.desc)

其生成的SQL（以MySQL方言为例）形如：

SELECT
    `t0`.`id` AS `c0`,
    `t0`.`name` AS `c1`,
    `t0`.`gender` AS `c2`,
    `t0`.`birthday` AS `c3`,
    `t0`.`id_number` AS `c4`
FROM `people` AS `t0`
ORDER BY `t0`.`id` ASC,
    `t0`.`name` DESC

支持的排序规则有：

排序规则
asc
desc
ascNullsFirst
ascNullsLast
descNullsFirst
descNullsLast

在生成MySQL等数据库方言时，会特殊处理含有NULLS的排序规则，不会生成错误的SQL。

sortBy不会改变查询结果类型

# 去重

使用distinct方法来对结果集进行去重：

val q = query[People].map(p => p.name).distinct

由于数据库对于DISTINCT和ORDER BY的相关限制，如果需要同时使用去重和排序，sortBy需要在distinct之后调用：

val q = query[People].map(p => p.name).distinct.sortBy(name => name.asc)

# 子查询

数据库通常支持三种子查询：

  1. WHERE和ON等子句中的谓词子查询；
  2. FROM和JOIN中的表子查询；
  3. SELECT子句中的标量子查询。

sqala对以上三种子查询均进行了支持。

# 谓词子查询

首先是IN、ANY、ALL、EXISTS等语义的子查询：

val q1 = query[A].filter: a =>
    a.x.in(query[B].map(b => b.x))

val q1 = query[A].filter: a =>
    a.x == any(query[B].map(b => b.x))

val q1 = query[A].filter: a =>
    a.x != all(query[B].map(b => b.x))

val q1 = query[A].filter: a =>
    exists(query[B].filter(b => b.x > 0))

以上类型的子查询除了exists和notExists外，均需要投影到与外侧表达式类型相符。

不使用以上语义的子查询：

val q1 = query[A].filter: a =>
    a.x == query[B].map(b => b.x).take(1)

val q2 = query[A].filter: a =>
    a.x > query[B].map(b => sum(b.x))

以上类型的子查询，除了需要与外侧表达式类型相符外，还有一个额外的限制，sqala必须检测到子查询仅返回一行数据，才能通过编译。

sqala从以下两种条件中判断子查询返回了一行数据：

  1. 子查询调用了take(1)；
  2. 子查询在没有调用groupBy的情况下，在map中返回了聚合函数。

# 表子查询

sqala支持将子查询放入表连接中，我们需要在join等方法后加上Query，使用表子查询的前提是子查询投影到了命名元组，此时我们可以使用.来调用子查询的字段：

val q = query[A].leftJoinQuery(
    query[B].map(b => (x = b.x, y = b.y))
).on((a, q) => a.x == q.x)

表子查询支持引用外侧字段，我们需要在join等方法后加上Lateral：

val q = query[A].leftJoinLateral(a =>
    query[B].filter(b => a.y == b.y).map(b => (x = b.x, y = b.y))
).on((a, q) => a.x == q.x)

使用此类方法时需要注意数据库版本是否已经支持了LATERAL语义。

query方法支持将一个返回命名元组的查询嵌套为子查询：

val q = query(query[A].map(a => (x = a.x, y = a.y)))

此方法可以将UNION等查询转变为子查询，从而使用SELECT语句独有的过滤、分组等功能。

使用*可以调用快速调用子查询的所有字段：

val q = query(query[A].map(a => (x = a.x, y = a.y))).map(_.*)

# 标量子查询

使用asExpr方法将返回了一行一列的查询转换为标量子查询，其可以放在map投影中：

val q = query[A].map: a =>
    (a.x, query[B].map(b => b.x).take(1).asExpr)

# 更好的可读性

将查询放在queryContext方法的作用域中，我们可以将非相关子查询单独提取出一个变量，以获取更好的可读性：

val q = queryContext:
    val subQuery = query[B].map(b => (x = b.x, y = b.y))

    query[A].leftJoinQuery(subQuery).on((a, q) => a.x == q.x)

# 集合操作

sqala支持使用union、unionAll、intersect、intersectAll、except、exceptAll等方法来处理集合查询，比如：

val q = queryContext:
    val q1 = query[People]
        .filter(p => p.id == 1)
        .map(p => (p.id, p.name))

    val q2 = query[People]
        .filter(p => p.id == 2)
        .map(p => (p.id, p.name))

    val q = q1 union q2

集合查询必须使用map投影。

由于UNION ALL与集合拼接语义一致，所以上面的unionAll可以简写成++：

val q = q1 ++ q2

处于集合操作两侧的查询，其返回类型必须列数量一致，且类型一一对应，假如有两个查询，分别返回(Option[Int], String, Option[Date])和(Int, Option[String], Date)，这样的两个查询调用集合操作将会返回(Option[Int], Option[String], Option[Date])。

# 从内存集合创建查询

使用values方法从内存中的集合创建查询，此查询可以使用投影过滤等操作，并可以与其他查询进行join或union等操作：

case class Entity(id: Int, name: String)
val list = List(Entity(1, "小黑"), Entity(2, "小白"))

val q = values(list).filter(e => e.id > 0).map(e => e.name)

这是使用数据库的VALUES语句实现的，使用时请注意当前数据库版本是否支持此语法。

# 递归查询

回想一下前文中表连接部分提到的部门表，如果我们想查询一整个部门树，可能需要发出多次查询，所幸，数据库支持CTE(Common Table Expression)功能，我们可以用来构建递归查询，而无需发出多次查询浪费数据库性能，sqala支持使用withRecursive方法创建递归查询：

val cte = queryContext {
    val baseQuery = query[Department]
        .filter(d => d.parentId == 0)
        .map(d => d.*)

    withRecursive(baseQuery): q =>
        val unionQuery = query[Department]
            .joinQuery(q)
            .on: (d, tmp) =>
                d.parentId == tmp.id
            .map((d, _) => d.*)
            
        q ++ unionQuery
}

其生成的SQL（以MySQL方言为例）形如：

WITH RECURSIVE `cte`(`c0`, `c1`, `c2`) AS (
    SELECT
        `t0`.`id` AS `c0`,
        `t0`.`parent_id` AS `c1`,
        `t0`.`name` AS `c2`
    FROM `department` AS `t0`
    WHERE `t0`.`parent_id` = 0
    UNION ALL
    SELECT
        `t1`.`id` AS `c0`,
        `t1`.`parent_id` AS `c1`,
        `t1`.`name` AS `c2`
    FROM `department` AS `t1`
        INNER JOIN `cte` ON `t1`.`parent_id` = `cte`.`c0`
)
SELECT
    `cte`.`c0` AS `c0`,
    `cte`.`c1` AS `c1`,
    `cte`.`c2` AS `c2`
FROM `cte`

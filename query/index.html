# 查询

在配置好元数据之后，我们就可以开始着手构建查询了，sqala的基础查询结构与Scala集合库非常相似：

import sqala.dsl.*

val q = queryContext {
    query[People].filter(_.id == 1).map(p => (id = p.id, name = p.name))
}

查询必须在queryContext方法中构建，该方法提供了用于构造查询的上下文。

# 构建查询

sqala的查询使用query方法构建：

val q = queryContext {
    query[People]
}

其生成的SQL（以MySQL方言为例）形如：

SELECT 
    `t0`.`id` AS `c0`, 
    `t0`.`name` AS `c1`, 
    `t0`.`gender` AS `c2`, 
    `t0`.`birthday` AS `c3`, 
    `t0`.`id_number` AS `c4` 
FROM `people` AS `t0`

查询返回的数据类型为：

val result: List[People] = db.fetch(q)

# 投影

使用map方法进行结果集的投影操作，其对应到SQL的SELECT子句，参数是一个表结构对象映射到表达式 (opens new window)及其组成的元组或命名元组（NamedTuple）：

val q = queryContext {
    query[People].map(p => (id = p.id, name = p.name))
}

其生成的SQL（以MySQL方言为例）形如：

SELECT 
    `t0`.`id` AS `c0`, 
    `t0`.`name` AS `c1`
FROM `people` AS `t0`

查询返回的数据为：

val result: List[(id: Int, name: String)] = db.fetch(q)

# 过滤

使用filter方法进行条件过滤，其对应到SQL的WHERE子句，参数是一个Boolean表达式 (opens new window)：

val id = 1
val q = queryContext {
    query[People].filter(_.id == id).map(p => (id = p.id, name = p.name))
}

其生成的SQL（以MySQL方言为例）形如：

SELECT 
    `t0`.`id` AS `c0`, 
    `t0`.`name` AS `c1` 
FROM `people` AS `t0` 
WHERE `t0`.`id` = ?

多次调用filter时将会使用AND来连接查询条件。

对于动态拼接的条件，sqala提供了filterIf方法，会在第一个参数值为true时使用AND才将条件拼接到查询中：

val id = 1
val name = "小黑"

val q = queryContext {
    query[People].filterIf(id > 0)(_.id == id).filterIf(name.nonEmpty)(_.name == name)
}

filter不会改变查询结果类型。

# 表连接

sqala支持使用join、leftJoin、rightJoin、fullJoin来处理表连接，并使用on来处理链接条件：

val q = queryContext {
    query[A]
        .join[B]
        .on((a, b) => a.id == b.id)
        .join[C]
        .on((a, b, c) => b.id == c.id)
}

on引用的参数类型为一个由表对象组成的元组，元组项的数量会根据连接的表数量变化。

此时返回的结果类型为：

val result: List[(A, B, C)] = db.fetch(q)

由于外连接会产生额外的空值，因此，sqala可以从表连接的信息中自动推断返回类型，如果我们把上面的查询改为先进行右连接再进行左连接：

val q = queryContext {
    query[A]
        .rightJoin[B]
        .on((a, b) => a.id == b.id)
        .leftJoin[C]
        .on((a, b, c) => b.id == c.id)
}

此时返回的结果类型变成了：

val result: List[(Option[A], B, Option[C])] = db.fetch(q)

这里显式写出了返回类型仅仅是为了演示，sqala会从查询中自动推断出合适的返回类型。

我们也可以方便地处理连接自身的情况，比如有一个存储部门树的表对象：

case class Department(
    id: Int,
    parentId: Int,
    name: String
)

我们可以这样来关联查询：

val q = queryContext {
    query[Department].leftJoin[Department].on((t1, t2) => t1.parentId == t2.id)
}

# 分组

使用groupBy配合聚合函数表达式进行分组操作：

val q = queryContext {
    query[People]
        .groupBy(p => (p.name, p.gender))
        .map((g, p) => (name = g._1, gender = g._2, count = count(), maxBirthday = max(p.birthday)))
}

其生成的SQL（以MySQL方言为例）形如：

SELECT 
    `t0`.`name` AS `c0`, 
    `t0`.`gender` AS `c1`, 
    COUNT(*) AS `c2`, 
    MAX(`t0`.`birthday`) AS `c3` 
FROM `people` AS `t0` 
GROUP BY `t0`.`name`, `t0`.`gender`

另外可以使用having过滤聚合出的数据：

val q = queryContext {
    query[People]
        .groupBy(p => (p.name, p.gender))
        .having(_ => count() > 1)
        .map((g, p) => (name = g._1, gender = g._2, count = count(), maxBirthday = max(p.birthday)))
}

# 排序

使用sortBy方法进行排序，参数是使用asc或desc方法生成的排序规则，多个sortBy会依次拼接：

val q = queryContext {
    query[People].sortBy(_.id.asc).sortBy(_.name.desc)
}

其生成的SQL（以MySQL方言为例）形如：

SELECT 
    `t0`.`id` AS `c0`, 
    `t0`.`name` AS `c1`, 
    `t0`.`gender` AS `c2`, 
    `t0`.`birthday` AS `c3`, 
    `t0`.`id_number` AS `c4` 
FROM `people` AS `t0` 
ORDER BY `t0`.`id` ASC, 
    `t0`.`name` DESC

sortBy不会改变查询结果类型

# 去重

使用distinct方法来对结果集进行去重：

val q = queryContext {
    query[People].map(p => (name = p.name)).distinct
}

# 子查询

sqala支持表达式 (opens new window)中的子查询和JOIN中的表子查询，首先来介绍表达式中的子查询：

返回一列数据的子查询可以作为表达式的一部分出现，比如：

val q = queryContext {
    val subQuery = query[People].map(p => max(p.birthday))

    query[People].filter(p => p.birthday < subQuery)
}

可以看到，在queryContext上下文中我们可以方便地分步构建查询，可以减少复杂查询的嵌套层数。

支持IN、ANY、SOME、ALL、EXISTS等子查询相关谓词：

queryContext {
    val subQuery = query[People].filter(_.name == "小黑").map(p => p.id)

    val q1 = query[People].filter(_.id == any(subQuery))

    val q2 = query[People].filter(_.id.in(subQuery))
}

需要在表达式左侧使用子查询，可以使用asExpr方法：

val q = queryContext {
    val subQuery = query[People].map(p => max(p.birthday))

    query[People].filter(p => subQuery.asExpr > p.birthday)
}

JOIN中的子查询使用join、leftJoin、rightJoin、fullJoin等方法，传入子查询，此时查询表达式会比较复杂，因此强烈推荐Scala3新的无括号语法，或者在queryContext中将子查询绑定到一个临时变量，代码会更加简洁：

val q = queryContext {
    query[A]
        .leftJoin:
            query[B].map: b =>
                (x = b.x, y = b.y)
        .on: (a, q) =>
            a.x == q.x
        .map: (a, q) =>
            (ay = a.x, by = q.y)
}

子查询使用命名元组（NamedTuple）给字段起名之后，就可以在后续使用字段名引用对应的字段。

上面leftJoin中的子查询只使用了一个表，且引入了表的全部字段，这种情况下我们可以使用.*这个简写形式来快速引用，而无需手写样板代码：

val q = queryContext {
    query[A]
        .leftJoin:
            query[B].map(_.*)
        .on: (a, q) =>
            a.x == q.x
        .map: (a, q) =>
            (ay = a.x, by = q.y)
}

子查询表中可以使用外侧表的字段：

val q = queryContext {
    query[A]
        .leftJoin: a =>
            query[B].map: b =>
                (bx = b.x, by = b.y, ax = a.x, ay = a.y)
        .on: (a, q) =>
            a.x == q.bx
        .map: (a, q) =>
            (ax = a.x, ay = a.y, bx = q.bx, by = q.by)
}

使用该类方法时需要注意数据库是否支持LATERAL功能，比如MySQL 8.0 版本以下不支持此功能。

query方法可以将一个UNION等非SELECT查询的形式转换为嵌套子查询，以方便使用SELECT查询独有的功能：

val q = queryContext {
    val innerQuery = query[A].map(a => (x = a.x, y = a.y))

    // 生成子查询
    query(innerQuery).filter(q => q.x == 1)
}

# 连接内存中的集合

sqala支持表连接中使用内存中的集合，这是使用数据库的VALUES功能实现的：

val list = List(A(1, "a"), A(2, "b"))

val q = queryContext {
    query[A]
        .join(list)
        .on((a, l) => a.x == l.x)
        .map((a, l) => (x = a.x, y = l.y))
}

其生成的SQL（以MySQL方言为例）形如：

SELECT 
    `t0`.`x` AS `c0`, 
    `t1`.`c1` AS `c1` 
FROM `a` AS `t0` 
    LEFT OUTER JOIN (
        VALUES ROW(1, 'a'), ROW(2, 'b')
    ) AS `t1`(`c0`, `c1`) 
    ON `t0`.`x` = `t1`.`c0`

使用时需要注意数据库是否支持此功能。

# 集合操作

sqala支持使用union、unionAll、intersect、intersectAll、except、exceptAll等方法来处理集合查询，比如：

queryContext {
    val q1 = query[People].filter(p => p.id == 1).map(p => (id = p.id, name = p.name))
    val q2 = query[People].filter(p => p.id == 2).map(p => (id = p.id, name = p.name))
    val q = q1 union q2
}

由于UNION ALL与集合拼接语义一致，所以上面的unionAll可以简写成++：

val q = q1 ++ q2

处于集合操作两侧的查询，其返回类型必须列数量一致，且类型一一对应，假如有两个查询，分别返回(Option[Int], String, Option[Date])和(Int, Option[String], Date)，这样的两个查询调用集合操作将会返回(Option[Int], Option[String], Option[Date])。

与JOIN类似，这些方法也可以与内存中集合互操作，同样使用VALUES功能实现：

queryContext {
    val q1 = query[People].filter(p => p.id == 1).map(p => (id = p.id, name = p.name))
    val l1 = List(A(1, "小黑"), A(2, "小白"))

    val q = q1 unionList l1
}

# 递归查询

回想一下前文中表连接部分提到的部门表，如果我们想查询一整个部门树，可能需要发出多次查询，所幸，数据库支持CTE(Common Table Expression)功能，我们可以用来构建递归查询，而无需发出多次查询浪费数据库性能，sqala支持使用withRecursive方法创建递归查询：

queryContext {
    val baseQuery = query[Department]
        .filter(d => d.parentId == 0)
        .map(_.*)

    val cte = withRecursive(baseQuery): q =>
        val unionQuery = query[Department]
            .join(q)
            .on((d, tmp) => d.parentId == tmp.id)
            .map((d, _) => d.*)
        q ++ unionQuery
}

其生成的SQL（以MySQL方言为例）形如：

WITH RECURSIVE `cte`(`c0`, `c1`, `c2`) AS (
    SELECT 
        `t0`.`id` AS `c0`, 
        `t0`.`parent_id` AS `c1`, 
        `t0`.`name` AS `c2` 
    FROM `department` AS `t0` 
    WHERE `t0`.`parent_id` = 0 
    UNION ALL     
    SELECT 
        `t1`.`id` AS `c0`, 
        `t1`.`parent_id` AS `c1`, 
        `t1`.`name` AS `c2` 
    FROM `department` AS `t1` 
        INNER JOIN `cte` ON `t1`.`parent_id` = `cte`.`c0`
) 
SELECT 
    `cte`.`c0` AS `c0`, 
    `cte`.`c1` AS `c1`, 
    `cte`.`c2` AS `c2` 
FROM `cte`

# 原生SQL

sqala支持使用原生SQL，我们通过sql字符串插值器即可创建原生SQL：

val x = 1
val y = "abc"

val nativeSql = sql"select * from t where x = $x and y = $y"

传入的变量会被处理为?，来避免SQL注入。

List类型变量会按元素数量生成(?, ?, ?)，但注意其不像sqala提供的查询构造器对空集合进行优化，原生SQL在传入空集合时可能会生成错误的SQL，请在传入时请检查集合是否为空。

原生SQL功能支持+运算，方便进行条件拼接：

val x = 1
val y = "abc"

var nativeSql = sql"select * from t where true "

if x > 0 then nativeSql += sql"and x = $x "

if y.nonEmpty then nativeSql += sql"and y = $y"

原生SQL功能失去了sqala的类型安全保障，除非需要数据库特殊方言，否则请尽量减少使用此功能。
